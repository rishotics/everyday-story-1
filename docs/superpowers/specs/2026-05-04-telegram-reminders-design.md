# Telegram Reminder Bot ‚Äî Design

**Date:** 2026-05-04
**Goal:** Make Everyday Story stickier by adding a Telegram bot that (a) nudges the user twice a day with an AI-personalized prompt and (b) lets the user reply to journal without opening the app.

## Summary

A Telegram bot DMs the user at **10:00 and 01:00 IST** with a personalized journaling prompt generated by Claude from recent stories. Any text the user sends back to the bot is appended to that day's story in MongoDB. The bot replies `Saved ‚úì`. No schema changes; reuses the existing `Story` collection. Implemented entirely as Next.js API routes triggered by Vercel Cron ‚Äî no new external services.

## Decisions

| Topic | Decision |
|---|---|
| Channel | Telegram bot DM (`t.me/every_day_story_rishotics_bot`) |
| Cadence | 10:00 IST + 01:00 IST, both fire unconditionally (no "already wrote" check) |
| Inbound handling | Any text message ‚Üí **appended** to today's story (newline-separated). No story yet ‚Üí create one. |
| Voice notes | Out of scope for v1 (text only) |
| Prompt style | AI-personalized: Claude reads recent stories, writes one specific question |
| Bot reply after save | Terse `Saved ‚úì` |
| Day boundary | **IST midnight** (Asia/Kolkata), so a 01:00 IST reply lands on the correct calendar day |
| App editor | Unchanged ‚Äî only the Telegram webhook appends; web `StoryEditor` keeps current create behavior |
| Scheduler | Vercel Cron |

## Environment variables

| Var | Purpose | Source |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | Bot API auth | BotFather (already created) |
| `TELEGRAM_CHAT_ID` | Numeric chat ID whitelist ‚Äî ignore all other senders | obtained via `getUpdates` after messaging the bot once |
| `TELEGRAM_WEBHOOK_SECRET` | Secret token Telegram echoes in `X-Telegram-Bot-Api-Secret-Token` on every update | generated during setup |
| `CRON_SECRET` | Vercel Cron auth bearer token | generated; Vercel auto-injects into cron requests |

Add all four to `.env.local` and Vercel project env. Update `.env.example` with placeholder names (no values).

## Component details

### `lib/istDate.ts`
- `istDayRange(now: Date)` ‚Üí `{ start: Date, end: Date }` covering the current IST calendar day expressed as UTC instants (IST = UTC+5:30, no DST).
- `istMidnightUTC(now: Date)` ‚Üí `Date` for the IST-midnight of the current IST day, stored as the story's `date` (consistent with how existing stories key on a day).

### `lib/telegram.ts`
- `sendMessage(chatId: string | number, text: string): Promise<void>` ‚Äî POST to `https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage` with `{ chat_id, text }`. Throws on non-2xx so callers can fall back.

### `lib/promptGenerator.ts`
- `generatePrompt(slot: "morning" | "night"): Promise<string>`
  - Connect DB, fetch last 20 stories sorted by `date` desc.
  - Call Anthropic (reuse `@anthropic-ai/sdk`, model consistent with existing `app/api/ai/route.ts`), `max_tokens` ~150.
  - System/meta-prompt: *"You craft a daily journal nudge. Read the user's recent stories. Write ONE specific, warm question (‚â§20 words) that picks up a thread from their entries. Slot=morning ‚Üí forward-looking; slot=night ‚Üí reflective. Output the question only, no preamble. At most one emoji at the end."*
  - On any error (no stories, API failure) ‚Üí return a random item from a static pool of ~5 prompts (morning/night variants).

### `app/api/telegram/cron/route.ts` (GET)
1. Verify `Authorization: Bearer ${CRON_SECRET}`; 401 if mismatch.
2. Read `slot` from query (`morning` default).
3. `prompt = await generatePrompt(slot)`.
4. `await sendMessage(TELEGRAM_CHAT_ID, prompt)`. On Telegram failure, retry once with a static fallback prompt.
5. Always return 200 (cron retries next day); log errors server-side.

### `app/api/telegram/webhook/route.ts` (POST)
1. Verify header `X-Telegram-Bot-Api-Secret-Token === TELEGRAM_WEBHOOK_SECRET`; 401 if mismatch.
2. Parse update. If `message.from.id !== Number(TELEGRAM_CHAT_ID)` ‚Üí return 200 silently (ignore strangers).
3. If `!message.text` ‚Üí `sendMessage("I can only save text right now.")`, return 200.
4. `connectDB()`. Compute `{ start, end } = istDayRange(now)`. Find a story with `date` in range.
5. If found ‚Üí `content = content + "\n\n" + message.text`, save. Else ‚Üí create `{ date: istMidnightUTC(now), content: message.text }`.
6. `await sendMessage(TELEGRAM_CHAT_ID, "Saved ‚úì")`. Return 200.
7. After step 1 passes, always return 200 even on internal error (prevents Telegram retry storms); log errors.

### `vercel.json`
Two cron entries (UTC):
- `30 4 * * *` ‚Üí `/api/telegram/cron?slot=morning` (10:00 IST)
- `30 19 * * *` ‚Üí `/api/telegram/cron?slot=night` (01:00 IST next IST day)

### `scripts/setup-telegram-webhook.ts`
One-time node script: calls Telegram `setWebhook` with the deployed URL `https://<domain>/api/telegram/webhook` and `secret_token = TELEGRAM_WEBHOOK_SECRET`. Documents how to fetch `TELEGRAM_CHAT_ID` via `getUpdates`.

## Error handling principles
- **Auth first, always 200 after:** both routes validate their secret up front (401 on failure). Once authenticated, they return 200 even on downstream errors so neither Vercel Cron nor Telegram retries aggressively.
- **Graceful prompt degradation:** Claude failure never blocks the nudge ‚Äî static fallback pool guarantees a message goes out.
- **Whitelist isolation:** only `TELEGRAM_CHAT_ID` can write stories; all other senders are silently ignored.

## Testing
- **`lib/istDate.ts`:** unit tests for the IST boundary ‚Äî e.g. 01:00 IST maps to the correct IST calendar day; UTC instants returned bracket that day.
- **Webhook:** simulate Telegram update payloads ‚Äî valid text appends; second message same day appends with newline; wrong chat ID ignored; bad secret ‚Üí 401; non-text ‚Üí polite reply.
- **Cron:** mock Anthropic + Telegram ‚Äî success path sends generated prompt; Claude failure falls back to static; bad `CRON_SECRET` ‚Üí 401.
- **Manual:** after deploy, message the bot and confirm the story appears in the app's Browse tab; verify both cron times fire (Vercel dashboard / temporary manual trigger).

## Out of scope (v1)
- Voice note transcription
- Inline buttons / conditional "already wrote today" logic
- AI follow-up questions after saving
- Multi-user / auth (remains a personal single-user app)
